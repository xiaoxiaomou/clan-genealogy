## 三、设计模式与交互参考

### 3.1 数据模型设计

所有族谱可视化库都围绕 **GEDCOM 标准** 设计数据结构。你的项目应支持：

```typescript
// 核型数据模型（参考 GEDCOM 5.5.1）
interface GedcomIndividual {
  id: string;           // @I1@ 格式的 GEDCOM ID
  name: string;         // 全名
  gender: 'M' | 'F' | 'U';
  birth?: { date: string; place: string };
  death?: { date: string; place: string };
  families_as_spouse: string[];    // 作为配偶的家庭 ID
  families_as_child: string[];     // 作为子女的家庭 ID
}

interface GedcomFamily {
  id: string;           // @F1@ 格式
  husband?: string;     // 夫 Individual ID
  wife?: string;        // 妻 Individual ID
  children: string[];   // 子女 Individual ID 列表
}
```

**你的项目当前模型**（Member + Relationship）已经与此类似，只需增加 GEDCOM 导出适配即可与所有开源工具互通。

### 3.2 色系与性别编码（行业标准）

| 性别 | 颜色 | 十六进制 | 用途 |
|------|------|---------|------|
| 男 | 蓝色系 | `#4A90D9` | 节点填充、边框 |
| 女 | 粉色系 | `#E8828D` | 节点填充、边框 |
| 未知 | 灰色系 | `#9CA3AF` | 中性节点 |
| 在世 | 实心 | — | 明亮显示 |
| 已故 | 半透明 | `opacity: 0.6` | 淡化处理 |
| 配偶连线 | 红色/粉色 | `#E8828D` | 婚姻关系线 |
| 亲子连线 | 黑色/灰色 | `#6B7280` | 血缘关系线 |

### 3.3 交互模式（行业标准）

| 交互 | 描述 | 来源参考 |
|------|------|---------|
| **点击聚焦** | 点击人物节点 → 树以该人物为中心重新布局 | Topola, Ancestry |
| **双击编辑** | 双击节点打开编辑面板 | Family Chart |
| **滚轮缩放** | 鼠标滚轮缩放整个树 | 所有方案标配 |
| **拖拽平移** | 拖拽空白区域平移视图 | 所有方案标配 |
| **悬停预览** | 悬停人物显示简要信息卡片 | Ancestry Hints |
| **右键菜单** | 右键节点显示操作菜单（编辑/删除/添加关系） | Family Chart Premium |
| **拖拽节点** | 拖拽人物节点调整位置 | React Flow, dTree |
| **框选** | Shift+拖拽框选多个节点 | Family Chart Premium |
| **搜索高亮** | 搜索姓名 → 高亮匹配节点并聚焦 | Ancestry, FamilySearch |

### 3.4 关键设计原则

1. **渐进式加载**：超过 100 个节点时应分片渲染，避免白屏
2. **视口适配**：默认视图应显示 3-5 代，用户可缩放至 10+ 代
3. **动画过渡**：点击聚焦时应有平滑的平移/缩放动画（Topola 参考：`transition: transform 0.5s ease`）
4. **离屏渲染**：大量节点时应只渲染视口可见区域（Virtual DOM / Canvas 方案）

## 四、中文传统谱式实现方案

### 4.1 三种传统谱式

| 谱式 | 布局 | 适合场景 | 实现难度 |
|------|------|---------|---------|
| **苏式** | 每页5代竖排，世系+人名竖列 | 正式家谱印刷 | ★★★★ |
| **欧式** | 横排表格，世代从左到右 | 快速编纂 | ★★★ |
| **宝塔式** | 金字塔展开，始祖在顶 | 展示型家谱 | ★★ |

### 4.2 推荐实现方式

```
苏式/欧式 → 使用 D3.js 树布局 + 自定义渲染器
宝塔式   → 使用 CSS flexbox/grid 布局（最简单）
导出     → jsPDF + html2canvas（网页截图 PDF）
```

1. **最佳集成方案**：`npm install topola` + `npm install reactflow`

4. **最大的差异化**：`Topola Viewer` 的沙漏图 (Hourglass)
   - 同时显示祖先和后代，视觉效果最震撼
   - 你的项目目前只有代际列表，沙漏图可大幅提升体验
