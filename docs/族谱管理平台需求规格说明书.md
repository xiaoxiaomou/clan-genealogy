## 1. 项目概述

### 1.1 项目简介

**项目名称**：族谱管理平台（Zupu）

**一句话描述**：面向中国家庭的现代化家族谱系管理平台，融合传统谱式格式与现代交互体验，支持族谱可视化、多媒体相册、亲属关系计算等核心功能。

### 1.2 项目背景

随着数字化时代的到来，传统族谱管理面临以下挑战：
- 纸质族谱不易保存、难以更新
- 家族成员分散各地，缺乏统一管理平台
- 传统格式与现代数字化工具脱节
- 家族历史和文化难以有效传承

本项目旨在通过数字化手段解决上述问题，为家族提供便捷的族谱管理和传承平台。

### 1.3 目标用户

| 用户群体 | 年龄范围 | 使用场景 | 主要需求 |
|---------|---------|---------|---------|
| 族长/族老 | 50-70岁 | 管理整族谱牒、审核修改、打印纸质谱书 | 传统谱式、大字界面、操作简便 |
| 中年族人 | 35-50岁 | 维护自家支系、上传照片、联络宗亲 | 数据维护、多媒体支持 |
| 年轻族人 | 18-35岁 | 浏览家族树、查看祖源故事、参与家族互动 | 可视化展示、社交互动 |

### 1.5 设计风格

**视觉定位**：现代东方美学 = Apple 玻璃质感 + 中国传统元素点缀

| 设计要素 | 规范 |
|---------|------|
| 基础风格 | 毛玻璃磨砂风格（glassmorphism） |
| 主色调（深色） | 深空黑底 `#0a0a0f` + 金色点缀 `#c9a84c` |
| 主色调（浅色） | 浅灰底 `#f5f5f7` + 暗红点缀 `#8b2500` |
| 男性配色 | 蓝色系 `#2563eb` / `#60a5fa` |
| 女性配色 | 粉色系 `#ec4899` / `#f472b6` |
| 字体 | 无衬线系统字体为主，标题可选书法字体 |
| 图标 | Lucide 简洁线性图标 |
| 响应式断点 | Mobile < 640px / Tablet 640-1024px / Desktop > 1024px |

### 6.3 数据库ER图设计

#### 6.3.1 ER图关系说明

| 关系类型 | 描述 | 表对 |
|---------|------|------|
| 1:N | 一对多 | families → members, family_events, family_albums, generation_rules, invitations |
| M:N | 多对多（中间表） | users ↔ families (family_members) |
| 1:N | 一对多（自关联） | family_branches → family_branches |
| 1:N | 一对多 | family_albums → family_photos |
| N:1 | 多对一（可选） | family_photos → members |

#### 6.3.2 完整建表语句（SQLite）

```sql
-- ============================================
-- 族谱管理平台 数据库建表脚本
-- 数据库: SQLite
-- 包含: 表结构、索引、外键约束、级联删除策略
-- ============================================

-- 1. 用户表
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(120) UNIQUE NOT NULL,
    password_hash VARCHAR(256) NOT NULL,
    display_name VARCHAR(80),
    avatar VARCHAR(256),
    is_admin BOOLEAN DEFAULT 0,

    FOREIGN KEY (branch_id) REFERENCES family_branches(id) ON DELETE SET NULL
);

-- 2. 族谱表
CREATE TABLE families (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name VARCHAR(100) NOT NULL,
    description TEXT,
    surname VARCHAR(50),
    origin VARCHAR(200),
    creator_id INTEGER NOT NULL,
    is_public BOOLEAN DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (creator_id) REFERENCES users(id) ON DELETE RESTRICT
);

-- 3. 族谱成员关联表（多对多中间表）
CREATE TABLE family_members (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    family_id INTEGER NOT NULL,
    user_id INTEGER NOT NULL,
    role VARCHAR(20) DEFAULT 'viewer',
    joined_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(family_id, user_id),
    FOREIGN KEY (family_id) REFERENCES families(id) ON DELETE CASCADE,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);

-- 4. 成员表
CREATE TABLE members (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    family_id INTEGER NOT NULL,
    name VARCHAR(50) NOT NULL,
    gender VARCHAR(10) DEFAULT 'unknown',
    birth_date VARCHAR(20),
    death_date VARCHAR(20),
    generation INTEGER,
    generation_name VARCHAR(50),
    bio TEXT,
    avatar VARCHAR(256),
    is_alive BOOLEAN DEFAULT 1,

    FOREIGN KEY (branch_id) REFERENCES family_branches(id) ON DELETE SET NULL
);

-- 5. 关系表
CREATE TABLE relationships (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    member_id INTEGER NOT NULL,
    related_member_id INTEGER NOT NULL,
    relationship_type VARCHAR(20) NOT NULL,
    member_name VARCHAR(50),
    related_member_name VARCHAR(50),

    FOREIGN KEY (member_id) REFERENCES members(id) ON DELETE CASCADE,
    FOREIGN KEY (related_member_id) REFERENCES members(id) ON DELETE CASCADE,
    CHECK (member_id != related_member_id)
);

    founder_id INTEGER,
    parent_branch_id INTEGER,
    sort_order INTEGER DEFAULT 0,
    member_count INTEGER DEFAULT 0,
    is_active BOOLEAN DEFAULT 1,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (family_id) REFERENCES families(id) ON DELETE CASCADE,
    FOREIGN KEY (founder_id) REFERENCES members(id) ON DELETE SET NULL,
    FOREIGN KEY (parent_branch_id) REFERENCES family_branches(id) ON DELETE RESTRICT
);

-- 7. 家族事件表
CREATE TABLE family_events (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    family_id INTEGER NOT NULL,
    title VARCHAR(200) NOT NULL,
    description TEXT,
    event_date VARCHAR(20),
    event_type VARCHAR(50) DEFAULT 'other',
    location VARCHAR(200),
    related_member_ids TEXT,
    images TEXT,
    created_by INTEGER NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (family_id) REFERENCES families(id) ON DELETE CASCADE,
    FOREIGN KEY (created_by) REFERENCES users(id) ON DELETE RESTRICT
);

-- 8. 家族相册表
CREATE TABLE family_albums (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    family_id INTEGER NOT NULL,
    name VARCHAR(100) NOT NULL,
    description TEXT,
    cover_photo_id INTEGER,
    created_by INTEGER NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (family_id) REFERENCES families(id) ON DELETE CASCADE,
    FOREIGN KEY (cover_photo_id) REFERENCES family_photos(id) ON DELETE SET NULL,
    FOREIGN KEY (created_by) REFERENCES users(id) ON DELETE RESTRICT
);

-- 9. 家族照片表
CREATE TABLE family_photos (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    album_id INTEGER NOT NULL,
    member_id INTEGER,
    title VARCHAR(200),
    description TEXT,
    file_path VARCHAR(256) NOT NULL,
    file_size INTEGER,
    mime_type VARCHAR(50),
    uploaded_by INTEGER NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (album_id) REFERENCES family_albums(id) ON DELETE CASCADE,
    FOREIGN KEY (member_id) REFERENCES members(id) ON DELETE SET NULL,
    FOREIGN KEY (uploaded_by) REFERENCES users(id) ON DELETE RESTRICT
);

-- 10. 辈分字派表
CREATE TABLE generation_rules (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    family_id INTEGER NOT NULL,
    generation INTEGER NOT NULL,
    character VARCHAR(10) NOT NULL,
    description VARCHAR(200),
    is_active BOOLEAN DEFAULT 1,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(family_id, generation),
    FOREIGN KEY (family_id) REFERENCES families(id) ON DELETE CASCADE
);

-- 11. 邀请码表
CREATE TABLE invitations (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    family_id INTEGER NOT NULL,
    code VARCHAR(64) UNIQUE NOT NULL,
    created_by INTEGER NOT NULL,
    role VARCHAR(20) DEFAULT 'viewer',
    max_uses INTEGER DEFAULT 0,
    use_count INTEGER DEFAULT 0,
    expires_at DATETIME,
    is_active BOOLEAN DEFAULT 1,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (family_id) REFERENCES families(id) ON DELETE CASCADE,
    FOREIGN KEY (created_by) REFERENCES users(id) ON DELETE RESTRICT
);

-- 12. 通知表
CREATE TABLE notifications (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    family_id INTEGER,
    title VARCHAR(100) NOT NULL,
    content TEXT,
    type VARCHAR(20) DEFAULT 'info',
    is_read BOOLEAN DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (family_id) REFERENCES families(id) ON DELETE CASCADE
);

-- 13. 审计日志表
CREATE TABLE audit_logs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    family_id INTEGER,
    user_id INTEGER,
    action VARCHAR(50) NOT NULL,
    entity_type VARCHAR(50) NOT NULL,
    entity_id INTEGER,
    description VARCHAR(500),
    ip_address VARCHAR(50),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (family_id) REFERENCES families(id) ON DELETE SET NULL,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE SET NULL
);
```

#### 6.3.4 级联删除策略汇总

| 父表 | 子表/关联表 | 删除行为 | 说明 |
|------|-----------|---------|------|
| users | family_members | CASCADE | 删除用户时，自动移除族谱成员关联 |
| users | members (founder_id) | SET NULL | 删除用户时，创始人置空 |
| users | family_events | RESTRICT | 删除用户时，如有关联事件则拒绝 |
| users | notifications | CASCADE | 删除用户时，自动删除其通知 |
| families | family_members | CASCADE | 删除族谱时，自动移除所有成员关联 |
| families | members | CASCADE | 删除族谱时，自动删除所有成员 |
| families | family_branches | CASCADE | 删除族谱时，自动删除所有分支 |
| families | family_events | CASCADE | 删除族谱时，自动删除所有事件 |
| families | family_albums | CASCADE | 删除族谱时，自动删除所有相册 |
| families | generation_rules | CASCADE | 删除族谱时，自动删除所有字派规则 |
| families | invitations | CASCADE | 删除族谱时，自动删除所有邀请码 |
| families | notifications | CASCADE | 删除族谱时，自动删除相关通知 |
| family_branches | members (branch_id) | SET NULL | 删除分支时，成员branch_id置空 |
| family_branches | family_branches (parent) | RESTRICT | 删除分支时，如存在子分支则拒绝 |
| family_branches | users (branch_id) | SET NULL | 删除分支时，用户branch_id置空 |
| family_albums | family_photos | CASCADE | 删除相册时，自动删除所有照片 |

| family_photos | family_albums (cover) | SET NULL | 删除照片时，相册封面置空 |

```json
{
  "username": "zhangsan",

  "password": "password123",
  "display_name": "张三"

```
- **响应**:
```json
{
  "message": "注册成功，请等待管理员审核",
  "user": { /* User对象 */ }
}
```

#### 7.1.2 用户登录
- **URL**: `POST /api/auth/login`
- **请求体**:
```json
{
  "username": "zhangsan",
  "password": "password123"
}
```
- **响应**:
```json
{
  "message": "登录成功",
  "user": { /* User对象 */ },
  "access_token": "eyJhbGciOiJIUzI1NiIs...",
  "refresh_token": "eyJhbGciOiJIUzI1NiIs..."
}
```

### 7.2 族谱管理接口

#### 7.2.1 获取族谱列表
- **URL**: `GET /api/family/`
- **认证**: 需要JWT
- **响应**:
```json
{
  "families": [ /* Family数组 */ ]
}
```

#### 7.2.2 创建族谱
- **URL**: `POST /api/family/`
- **认证**: 需要JWT (admin)

#### 7.8.1 获取所有配置

- **权限**: admin
- **响应**:
```json
{
  "configs": [
    {
      "id": 1,
      "category": "user",
      "key": "user.allow_register",

  ]
}
```

#### 7.8.2 按分类获取配置
- **URL**: `GET /api/admin/config/category/<category>`
- **权限**: admin
- **参数**: category 为 user/family/content/notification/ui
- **响应**: 同 7.8.1

#### 7.8.3 更新单个配置
- **URL**: `PUT /api/admin/config/<key>`
- **权限**: admin
- **请求体**:
```json
{
  "value": "false",
  "reason": "维护期间暂停注册"
}
```
- **响应**:
```json
{
  "message": "更新成功",
  "config": { ... }
}
```

#### 7.8.4 批量更新配置
- **URL**: `PUT /api/admin/config/batch`
- **权限**: admin
- **请求体**:
```json
{
  "updates": [
    { "key": "user.allow_register", "value": "false" },
```
