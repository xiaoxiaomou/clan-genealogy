## 报告漏洞

我们非常重视安全问题。**请不要通过公开 Issue 报告安全漏洞**，以免在修复前被恶意利用。

请通过以下方式私密报告：

- 使用 GitHub 的 **[Private vulnerability reporting](../../security/advisories/new)**（推荐）；或
- 发送邮件至维护者（请在仓库主页 / `LICENSE` 中获取联系方式）。

报告时请尽量包含：

- 漏洞类型与影响范围；
- 复现步骤或 PoC；
- 受影响的版本 / 环境；
- 可能的修复建议（如有）。

我们会在收到报告后尽快确认，并在修复发布前与你保持沟通。感谢你的负责任披露。

## 安全最佳实践（部署方）

- 生产环境务必设置强随机的 `SECRET_KEY` 与 `JWT_SECRET_KEY`（应用检测到默认值会拒绝启动）。
- 通过 `CORS_ORIGINS` 严格限制允许的前端来源。
- 使用 HTTPS 反向代理，妥善保管数据库凭据与 `.env` 文件。
- 定期备份数据库与上传文件。
